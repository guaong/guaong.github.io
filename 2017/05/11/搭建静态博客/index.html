搭建静态博客

2017-05-11
#工具


写在前面，该静态博客使用最常见的方法，即hexo+github page。 作为一个新手，在经历了两次失败之后终于成功。
本文主要记录如何搭建以及在搭建过程中遇到的问题如何解决。

正文：
准备工作：首先需要node.js，Git 至于下载的方式自行查找
当准备好上述两个后，就可以准备下载hexo了
安装hexo可以通过Git或者window自带的cmd输入：
$ npm install -g hexo-cli
我没有使用Git，我用的是cmd
等待安装完成后开始建站
1.新建一个文件夹，例“test”
2.鼠标右键“test”文件夹，打开git bash（该目的只是打开bash目录直接是你创建的位置）
3.输入：
$ hexo init testtest为你创建的文件夹
4.执行：
$ npm install
此时hexo暂时告一段落
准备github page
5.new repository
6.repository name，名字一定要是xx.github.io格式
7.勾选Initialize this repository with a README
8.create repository
项目创建完成
9.进入项目找到settings，点击进入
10.找到Source 默认none，选择master branch，然后save。如果发现不能勾选master branch，参见第7步，没有勾选
将hexo部署到github
11.打开刚刚创建的test文件夹，找到_config.yml，打开
12.找到其中的deploy，type设为git
13.repository设为git@github.com:你的github名字/xx.github.io.git注意如果你不知道repository该填什么，打开github找到该仓库，查找ssh
14.还是在_config.yml中，找到#URL
15.url填写你创建的仓库名即xx.github.io/xx.github.io注意，我不知道为什么要这么写，但是之前测试中没有写后面重复的那一编不显示主题,注意如果你也出现仓库网址为aa.github.io/xx.github.io的话url应填写aa.github.io/xx.github.io/xx.github.io
16.root填写xx.github.io
17.在Git中输入$ hexo deploydeploy可用d代替,若此处出错有可能是ssh key的事情，输入$ ssh-keygen -t rsa -C “github邮箱”一直回车最后会出现key，复制下来，打开github，修改test仓库的ssh
18.你可以通过Git输入$ hexo server在localhost:4000预览样式主题，ctrl+c停止server
19.接着输入$ hexo g
20.$ hexo d成功后打开连接即可看到效果
关于主题修改
21.从hexo官网下载主题,修改_config.yml文件中theme，将默认的文件夹名字改为你下载的主题文件夹名字即可。
关于信息更新
22.每次都要执行$ hexo clean$ hexo g$ hexo d这三步用于更新网页内容
关于新建博客
23.Git中输入$ hexo new 文件名创建，在test/source/_posts中找到文件名.md,打开修改即可,如何输入自行查询markdown
